static generate_random_hash() → str

Generates a random hash for a State when there is not enough data for a real one.

This may seem contra-intuitive, but works, because states are checked for similarity via the hash. We want to consider only states, that have enough exploration data when running the state collapsing pipeline and this method ensures that not properly explored states won’t be collapsed if they seem alike in the beginning. (i.e. this ensures lower false positive detections in the early exploration process.)
